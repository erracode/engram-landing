## OpenCode

**Prerequisite**: Install the `engram` binary first via [Homebrew](#homebrew-macos--linux), [Windows binary](#windows), [binary download](#download-binary-all-platforms), or [source](#install-from-source-macos--linux).

### Recommended: Full setup

```bash
engram setup opencode
```

This does two things:
1. Copies the plugin to `~/.config/opencode/plugins/engram.ts`
2. Adds the `engram` MCP server entry to `opencode.json`

The plugin also needs the HTTP server running:

```bash
engram serve &
```

**Windows**: Run `Start-Process engram -ArgumentList "serve" -WindowStyle Hidden` in a separate terminal.

### Alternative: Manual MCP-only setup

Add to `opencode.json`:

```json
{
  "mcp": {
    "engram": {
      "type": "local",
      "command": ["engram", "mcp"],
      "enabled": true
    }
  }
}
```

## Claude Code

### Option A: Plugin via marketplace (Recommended)

```bash
claude plugin marketplace add Gentleman-Programming/engram
claude plugin install engram
```

The plugin registers the MCP server automatically.

### Option B: Plugin via `engram setup`

```bash
engram setup claude-code
```

During setup, you'll be asked whether to add engram tools to `~/.claude/settings.json` permissions allowlist.

### Option C: Bare MCP

Add to `.claude/settings.json`:

```json
{
  "mcpServers": {
    "engram": {
      "command": "engram",
      "args": ["mcp"]
    }
  }
}
```

## Gemini CLI

```bash
engram setup gemini-cli
```

This does three things:
- Registers `mcpServers.engram` in `~/.gemini/settings.json`
- Writes `~/.gemini/system.md` with the Memory Protocol
- Ensures `~/.gemini/.env` contains `GEMINI_SYSTEM_MD=1`

### Manual alternative

Add to `~/.gemini/settings.json`:

```json
{
  "mcpServers": {
    "engram": {
      "command": "engram",
      "args": ["mcp"]
    }
  }
}
```

Or via CLI:

```bash
gemini mcp add engram engram mcp
```

## Codex

```bash
engram setup codex
```

This does three things:
- Registers `[mcp_servers.engram]` in `~/.codex/config.toml`
- Writes `~/.codex/engram-instructions.md` with the Memory Protocol
- Writes `~/.codex/engram-compact-prompt.md` for compaction recovery

### Manual alternative

Add to `~/.codex/config.toml`:

```toml
model_instructions_file = "~/.codex/engram-instructions.md"
experimental_compact_prompt_file = "~/.codex/engram-compact-prompt.md"

[mcp_servers.engram]
command = "engram"
args = ["mcp"]
```

## VS Code (Copilot / Claude Code Extension)

### Option A: Workspace config

Add to `.vscode/mcp.json`:

```json
{
  "servers": {
    "engram": {
      "command": "engram",
      "args": ["mcp"]
    }
  }
}
```

### Option B: User profile

1. Open Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`)
2. Run **MCP: Open User Configuration**
3. Add the same `engram` server entry to VS Code User `mcp.json`

### Option C: CLI one-liner

```bash
code --add-mcp '{"name":"engram","command":"engram","args":["mcp"]}'
```

### Adding the Memory Protocol

Create a `.instructions.md` file in the VS Code User `prompts/` folder:

```markdown
## Memory
You have access to Engram persistent memory via MCP tools (mem_save, mem_search, mem_session_summary, etc.).
- Save proactively after significant work — don't wait to be asked.
- After any compaction or context reset, call `mem_context` to recover session state before continuing.
```

## Surviving Compaction (Recommended)

When your agent compacts (summarizes long conversations to free context), it might forget about Engram. To make memory resilient, add this to your agent's system prompt:

```markdown
## Memory
You have access to Engram persistent memory via MCP tools (mem_save, mem_search, mem_session_summary, etc.).
- Save proactively after significant work — don't wait to be asked.
- After any compaction or context reset, call `mem_context` to recover session state before continuing.
```

This is the **nuclear option** — system prompts survive everything, including compaction.
