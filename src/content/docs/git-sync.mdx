# Git Sync

Share memories across machines. Uses compressed chunks — no merge conflicts, no huge files.

---

## Quick Start

```bash
# Export new memories as compressed chunk
engram sync

# Add to git and commit
git add .engram/ && git commit -m "sync engram memories"

# Import on another machine
engram sync --import

# Check sync status
engram sync --status
```

## How It Works

<div class="not-prose text-center my-6">

```
Machine A (Development)                      Machine B (Home)

┌─────────────────┐                          ┌─────────────────┐
│  Engram DB      │                          │  Engram DB      │
│  (~/.engram/    │                          │  (~/.engram/    │
│   engram.db)    │                          │   engram.db)    │
└────────┬────────┘                          └────────┬────────┘
         │                                           │
         │ engram sync                               │ engram sync --import
         ▼                                           ▼
┌─────────────────┐                          ┌─────────────────┐
│  .engram/       │                          │  .engram/       │
│  ├── chunk_001 │                          │  ├── chunk_001 │
│  │   .zst       │────── git push ────────▶ │   .zst         │
│  ├── chunk_002 │                          │   .zst         │
│  │   .zst       │                          │   .zst         │
│  └── manifest   │                          │  └── manifest   │
│      .json      │                          │      .json      │
└─────────────────┘                          └─────────────────┘
```

</div>

**Compressed Chunks:**
- Each chunk is compressed with Zstandard (.zst)
- Typically 10-50KB per chunk
- No merge conflicts because chunks are append-only
- Manifest tracks which chunks exist on each machine

## Exporting Memories

### Single Machine

Export new memories from the current directory's project:

```bash
engram sync
```

This creates:
- `.engram/chunk_XXX.zst` — Compressed memory data
- `.engram/manifest.json` — List of all chunks and metadata

### All Projects

Export memories from ALL projects (ignore directory filter):

```bash
engram sync --all
```

Useful when switching between repositories and you want to sync all recent memories.

### Manual Export

Export to a specific file:

```bash
engram export memories.json
```

This creates a full JSON export (not compressed, not chunked).

---

## Importing Memories

### From Git Repository

Import chunks that exist on the remote:

```bash
engram sync --import
```

This:
- Checks manifest for missing chunks
- Downloads missing chunks from git
- Merges new memories into local database
- Updates local manifest

### From Archive

Import from a specific directory:

```bash
engram sync --import --dir /path/to/.engram
```

Useful when manually transferring chunks via USB or other means.

### Status Overview

Check what chunks are synced:

Output example:

```

Remote chunks: 4
Unsynced: 1 (chunk_005.zst)
Total local memories: 127
Total remote memories: 123
```

### Compare Databases

Compare local and remote databases:

```bash
engram sync --compare
```

Shows:
- Memories only on local
- Memories only on remote
- Conflicts (same observation, different content)

## Conflict Resolution

Git Sync uses **append-only chunks** to avoid conflicts:

- Each chunk is immutable once created
- Chunks are never overwritten
- Conflicts happen only if the same observation is modified on both machines

### Resolving Conflicts

If conflicts are detected:

```
Conflict detected: observation_id=42
  Local: "Fixed bug in auth middleware"
  Remote: "Fixed auth nil panic in middleware"

Which version to keep?
[1] Local version
[2] Remote version
[3] Keep both (merge)
[q] Quit (keep local)
```

Choose:
- **1** — Use local version
- **2** — Use remote version  
- **3** — Merge both (creates revision)
- **q** — Keep local, sync later

## Git Workflow

### Recommended Workflow

1. **Start working** — Make changes in your agent session
2. **Save memories** — Agent calls `mem_save` proactively
3. **Sync locally** — `engram sync` creates new chunks
4. **Commit to git** — `git add .engram/ && git commit -m "sync memories"`
5. **Push to remote** — `git push`
6. **On another machine** — `git pull && engram sync --import`

### .gitignore

Don't add `.engram/` to `.gitignore` — you want to sync it!

Add this instead to avoid committing generated files:

```gitignore
# But keep .engram for sync
!.engram/
# Ignore build artifacts
dist/
node_modules/
```

### Automated Sync

Add to your `Makefile` or `.github/workflows`:

```yaml
name: Memory Sync
on:
  push:
    branches: [main]
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install Engram
        run: |
          brew install gentleman-programming/tap/engram
      - name: Sync memories
        run: |
          engram sync
          git add .engram/
          git commit -m "sync memories [skip ci]" || true
          git push
```

## Manual Chunk Management

### List Chunks

See all chunks in the current project:

```bash
engram projects list
```

Shows:
- Project name
- Number of observations
- Number of sessions
- Number of chunks

### Remove Old Chunks

Clean up old chunks that are no longer referenced:

```bash
engram sync --cleanup
```

This removes chunks that aren't in the manifest.

### Export All Projects

Export memories from ALL projects (not just current directory):

```bash
engram sync --all
```

Creates chunks for all projects in a single commit.

## Tips

- **Commit frequently** — Sync after each significant session
- **Pull before sync** — `git pull && engram sync --import`
- **Use descriptive commits** — "sync memories: auth middleware fix"
- **Don't edit chunks** — Treat them as immutable
- **Test import** — `engram sync --import` before pushing to remote
- **Backup manifest** — Keep a backup of `.engram/manifest.json`

## Advanced: Custom Storage

Use a different remote storage (not Git):

```bash
# Upload chunks to S3
aws s3 cp .engram/ s3://my-bucket/engram/

# Download chunks from S3
aws s3 sync s3://my-bucket/engram/ .engram/
engram sync --import
```

Or use Dropbox, Google Drive, or any cloud storage — as long as you can:
1. Upload chunks
2. Download chunks
3. Keep manifest synchronized

## FAQ

**Q: Will I lose memories if I delete a chunk?**

A: No. The manifest tracks which chunks are needed. If a chunk is deleted from git but not from your local `.engram/` folder, the memory is still accessible locally.

**Q: Can I sync to multiple machines?**

A: Yes! Git Sync is designed for this. Each machine keeps its own local database and syncs chunks via git.

**Q: What if I have network issues?**

A: Git Sync works offline. You can sync chunks locally, then push to git when you have internet access.

**Q: How big are the chunks?**

A: Typically 10-50KB each. A full session with 20-30 memories might create 1-2 chunks.

**Q: Can I export to JSON?**

A: Yes, use `engram export memories.json` for a full JSON export.
