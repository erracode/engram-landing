### Architecture

**Core Components:**

- **MCP Server** — 15 tools for memory management
- **SQLite + FTS5** — Persistent storage with full-text search
- **HTTP API** — REST API (port 7437)
- **CLI** — Command-line interface
- **TUI** — Interactive terminal UI
- **Git Sync** — Cross-machine sync with compressed chunks

**Session Lifecycle:**
1. Agent works and saves memories proactively
2. Session ends with summary
3. Next session injects previous context
4. Memories persist across sessions

[Full architecture guide →](https://github.com/Gentleman-Programming/engram/blob/main/docs/ARCHITECTURE.md)

<div id="mcp-tools">

### MCP Tools

**15 Tools for Memory Management:**

- `mem_save` — Save structured observations
- `mem_search` — Full-text search with FTS5
- `mem_timeline` — Chronological context
- `mem_context` — Recent session context
- `mem_session_summary` — End-of-session summary
- `mem_update` — Update existing observations
- `mem_delete` — Soft or hard delete
- `mem_get_observation` — Get full content
- `mem_suggest_topic_key` — Suggest stable topic keys
- `mem_stats` — Memory statistics
- `mem_session_start` — Register session start
- `mem_session_end` — Mark session complete
- `mem_save_prompt` — Save user prompts
- `mem_capture_passive` — Extract learnings from text
- `mem_merge_projects` — Merge project variants

[Complete MCP tools reference →](https://github.com/Gentleman-Programming/engram/blob/main/docs/ARCHITECTURE.md#mcp-tools)

<div id="git-sync">

### Git Sync

**Share memories across machines:**

```bash
engram sync                    # Export chunks
git add .engram/ && git commit -m "sync memories"
engram sync --import           # Import on another machine
```

**Benefits:**
- No merge conflicts (append-only chunks)
- Compressed storage (~10-50KB per chunk)
- Full-text search preserved
- Soft-delete support

[Git Sync guide →](https://github.com/Gentleman-Programming/engram/blob/main/DOCS.md#git-sync)

### Comparison

### Memory Protocol

The Memory Protocol tells agents when to save and search memories:

```markdown
## Memory
You have access to Engram persistent memory via MCP tools (mem_save, mem_search, mem_session_summary, etc.).
- Save proactively after significant work — don't wait to be asked.
- After any compaction or context reset, call `mem_context` to recover session state before continuing.
```

[Full Memory Protocol →](https://github.com/Gentleman-Programming/engram/blob/main/DOCS.md#memory-protocol-full-text)

### CLI Reference

```
engram setup [agent]      Install/setup agent integration
engram serve [port]       Start HTTP API server
engram mcp                Start MCP server
engram tui                Launch interactive terminal UI
engram search <query>     Search memories
engram save <title> <msg> Save a memory
engram timeline <obs_id>  Chronological context
engram context [project]  Recent context
engram stats              Memory statistics
engram export [file]      Export to JSON
engram import <file>      Import from JSON
engram sync               Export chunks to .engram/
engram projects list      Show project stats
engram version            Show version
```

[Full CLI reference →](https://github.com/Gentleman-Programming/engram/blob/main/DOCS.md#cli-reference)
